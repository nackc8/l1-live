[1mNAME[0m[m
       ps - report a snapshot of the current processes.[m
[m
[1mSYNOPSIS[0m[m
       [1mps[0m [[4moptions[24m][m
[m
[1mDESCRIPTION[0m[m
       [1mps[0m displays information about a selection of the active processes.  If you want a[m
       repetitive update of the selection and the displayed information, use [1mtop[0m instead.[m
[m
       This version of [1mps[0m accepts several kinds of options:[m
[m
       1   UNIX options, which may be grouped and must be preceded by a dash.[m
       2   BSD options, which may be grouped and must not be used with a dash.[m
       3   GNU long options, which are preceded by two dashes.[m
[m
       Options of different types may be freely mixed, but conflicts can appear.  There are[m
       some synonymous options, which are functionally identical, due to the many standards[m
[7m Manual page ps(1) line 1 (press h for help or q to quit)[27m[K[K       and [1mps[0m implementations that this [1mps[0m is compatible with.[m
[m
       Note that [1mps[0m [1m-aux[0m is distinct from [1mps[0m [1maux[0m.  The POSIX and UNIX standards require that[m
       [1mps[0m [1m-aux[0m print all processes owned by a user named [4mx[24m, as well as printing all processes[m
       that would be selected by the [1m-a[0m option.  If the user named [4mx[24m does not exist, this [1mps[0m[m
       may interpret the command as [1mps[0m [1maux[0m instead and print a warning.  This behavior is[m
       intended to aid in transitioning old scripts and habits.  It is fragile, subject to[m
       change, and thus should not be relied upon.[m
[m
       By default, [1mps[0m selects all processes with the same effective user ID (euid=EUID) as the[m
       current user and associated with the same terminal as the invoker.  It displays the[m
       process ID (pid=PID), the terminal associated with the process (tname=TTY), the[m
       cumulated CPU time in [DD-]hh:mm:ss format (time=TIME), and the executable name[m
       (ucmd=CMD).  Output is unsorted by default.[m
[m
       The use of BSD-style options will add process state (stat=STAT) to the default display[m
       and show the command args (args=COMMAND) instead of the executable name.  You can[m
       override this with the [1mPS_FORMAT[0m environment variable. The use of BSD-style options[m
       will also change the process selection to include processes on other terminals (TTYs)[m
       that are owned by you; alternately, this may be described as setting the selection to[m
[7m Manual page ps(1) line 21 (press h for help or q to quit)[27m[K[K       be the set of all processes filtered to exclude processes owned by other users or not[m
       on a terminal.  These effects are not considered when options are described as being[m
       "identical" below, so [1m-M[0m will be considered identical to [1mZ[0m and so on.[m
[m
       Except as described below, process selection options are additive.  The default[m
       selection is discarded, and then the selected processes are added to the set of[m
       processes to be displayed.  A process will thus be shown if it meets any of the given[m
       selection criteria.[m
